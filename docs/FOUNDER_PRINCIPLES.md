# Founder Principles

Business OS is built from customer reality, not competitor checklists.

## Customer First

Every feature must solve a real customer problem.

Start with the restaurant owner, cafe owner, street food seller, driver, HR
admin, family business, and small office worker who needs to finish work today.

## Simple Wins

Simple workflows beat clever systems.

If a busy customer cannot understand what to do next, the product is not ready.

## Trust Through Transparency

Show how calculations, reports, imports, exports, and AI suggestions work.

Trust grows when customers can understand the result.

## Respect Before Replace

Respect users' existing workflow.

If Excel helps customers, Excel stays.

If PDF helps customers communicate, PDF stays.

If accounting software or government systems already work for the customer,
Business OS should cooperate with them.

## Open by Default

Business OS should support open and familiar formats.

Data should move in and out without trapping customers.

## Small Business First

Design for Thai SMEs before enterprise complexity.

Business OS exists for people who are busy running businesses, not learning
software.

## Build Workflows, Not Feature Lists

We build workflows, not feature lists.

Never build features because competitors have them.

Technology exists to reduce stress, not create it.

## Feature Rule

Before building, ask:

- Does this solve a real customer problem?
- Does it save time?
- Does it reduce mistakes?
- Does it reduce stress?
- Can the user understand it without training?
- Does it respect existing workflows?

If the answer is mostly no, do not build it yet.

# Principles

## Product Principles

- Simple beats complex.
- Reliable beats clever.
- Helpful beats fancy.
- Trust beats feature count.
- Customer problems beat technical curiosity.
- User feedback beats assumptions.
- Long-term thinking beats short-term hacks.
- Configurability beats hardcoding.
- Maintainability beats speed.
- Explain before impress.
- Human first.
- Natural language beats technical terminology.
- Thai-first user experience.
- Every feature must solve a real customer problem.

## Engineering Principles

- Everything configurable.
- Everything modular.
- Everything reusable.
- Everything documented.
- Everything tested.
- Everything production-ready.
- Everything versioned.
- Everything maintainable.
- No hardcoded company rules.
- No duplicated business logic.
- Shared Core.
- Shared Authentication.
- Shared Authorization.
- Shared Configuration.
- Shared Reporting.
- Shared Database.
- Shared Notifications.
- Shared Localization.
- Shared Theme.
- Shared AI.
- Shared Analytics.
- Shared Audit.
- Shared Security.

## Release Principles

Every release must improve at least one of:

- Save time.
- Reduce mistakes.
- Reduce stress.
- Increase confidence.
- Improve business visibility.

If a feature does not improve one of these, it should not ship.

## Decision Principles

Before building, ask:

- Does this solve a real customer problem?
- Would a restaurant owner understand this?
- Would a coffee shop owner use this tomorrow?
- Does it reduce stress?
- Does it save time?
- Does it reduce mistakes?
- Can the business rule be configured?
- Can another Assistant reuse this?
- Can every business configure it?

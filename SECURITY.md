# Security Policy

Business OS products may handle sensitive business information, including
employee records, payroll calculations, expenses, reports, inventory, and tax
preparation data.

Protecting customer trust is mandatory.

## Supported Versions

TimeTrack Pro is currently in public beta preparation. Security fixes should
target the active beta branch unless the repository owner defines another
release policy.

## Reporting a Security Issue

Report suspected security issues privately to the repository owner or founder.

Do not publish exploit details publicly before the issue is reviewed and a fix
plan is defined.

## Security Principles

- User business data belongs to the user.
- Do not expose private data unnecessarily.
- Do not log sensitive business information without a clear reason.
- Do not invent or hide financial calculations.
- Validate backup and restore workflows before trusting imported data.
- Prefer simple, auditable security decisions over clever shortcuts.

## AI and Data

AI features must never invent financial facts or fabricate calculations.

AI recommendations must explain why they were made and must not imply legal,
tax, or financial certainty without verified rules and user-provided data.
